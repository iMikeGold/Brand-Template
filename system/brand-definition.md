# BRAND DEFINITION — OURGANI ECOSYSTEM

## Overview

Every repository created from this template represents a single "Brand Node" within the Ourgani ecosystem.

A Brand Node is a self-contained execution unit that delivers a specific function, service, product, or creative output while inheriting structural rules from the Ourgani System.

---

## Core Identity Model

Each brand consists of four layers:

### 1. System Layer (Inherited)
- Provided by Ourgani System
- Defines architecture rules
- Not modified by individual brands

### 2. Brand Layer (This Repository)
- Defines identity, purpose, and output
- Contains services, content, or products
- Owns its own deployment

### 3. Interface Layer (main-site/)
- Public-facing website structure
- Always deployed as the root of the site
- Must follow standard layout conventions

### 4. Registry Layer
- Machine-readable metadata (JSON)
- Used for system-wide coordination
- Ensures interoperability between brands

---

## Brand Purpose Rule

A brand MUST exist to fulfil one clear function:

- Commerce (selling or distributing)
- Intelligence (analysis, consultancy, systems thinking)
- Creative Output (media, music, design, culture)
- Infrastructure (tools, platforms, systems)
- Community / Education (learning, guidance, engagement)

No brand should exist without a defined functional category.

---

## System Relationship

All brands:
- Inherit structure from Ourgani System
- Operate under ProtoSynthesis logic (external system layer)
- Are registered in the central ecosystem registry
- Are interoperable through system standards

---

## Independence Rule

Each brand:
- Owns its own repository
- Owns its own deployment
- Owns its own domain mapping

However:
- Does NOT define system architecture
- Does NOT redefine ecosystem rules
- Does NOT break structural conventions
